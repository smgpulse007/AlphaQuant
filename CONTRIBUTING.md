# Contributing to AlphaQuant

Thanks for helping improve this project.

## Ways to contribute

- report bugs and missing features with clear reproduction steps;
- suggest architecture or workflow improvements;
- improve documentation, examples, or demo assets;
- submit pull requests for fixes and enhancements.

## Development workflow

1. Fork the repository and create a feature branch.
2. Install the local dependencies listed in the README.
3. Make focused changes and document any new behavior.
4. Verify the relevant command or test path before opening a PR.
5. Open a pull request with a short summary, screenshots or GIFs when relevant, and any notes about validation.

## Pull request expectations

- keep the scope focused;
- explain why the change is needed;
- include validation notes or commands used to verify the fix;
- update docs or demos when behavior changes.

## Code of conduct

Please review [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before participating.
